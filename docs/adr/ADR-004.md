# ADR-004: Prototipo mobile hibrido con Ionic

**Estado:** Aceptado  
**Fecha:** 2026-05-17  
**Decisor:** Equipo de Desarrollo  

## Contexto

La entrega final solicita construir un prototipo de aplicacion web/mobile utilizando un framework de desarrollo hibrido iOS/Android. El proyecto ya cuenta con un MVP web en Flask para gestionar canchas y reservas, por lo que la evolucion mobile debe reutilizar la mayor cantidad posible de logica existente.

La solucion debe ser simple, academica y demostrable, priorizando funcionamiento minimo viable por sobre arquitectura avanzada.

## Opciones Consideradas

1. **Ionic React con Capacitor** - Framework hibrido basado en tecnologias web.
2. **Flutter** - Framework multiplataforma basado en Dart.
3. **Aplicacion nativa Android** - Desarrollo especifico para Android.
4. **Solo mockup navegable** - Prototipo visual sin integracion con backend.

## Decisión

**Usar Ionic React con Capacitor para el prototipo mobile hibrido.**

## Rationale

1. **Continuidad tecnica:** Ionic reutiliza HTML, CSS, TypeScript y React, tecnologias cercanas al enfoque web del proyecto.
2. **Compatibilidad hibrida:** Capacitor permite preparar builds Android/iOS desde una misma base de codigo.
3. **Menor curva de aprendizaje:** El equipo puede explicar la solucion desde conceptos web ya trabajados.
4. **Integracion simple:** La app consume endpoints JSON expuestos por Flask sin reemplazar el backend existente.
5. **Alcance academico:** Permite demostrar navegacion mobile, listado y creacion de reservas sin sobreingenieria.

## Consecuencias

**Positivas:**
- Desarrollo rapido del prototipo.
- Reutilizacion del backend y reglas de negocio existentes.
- Compatibilidad con navegador, Android e iOS mediante Capacitor.
- Codigo separado en `mobile-ionic`, sin afectar la interfaz administrativa.

**Negativas:**
- Requiere mantener dependencias Node/Ionic ademas del entorno Python.
- El rendimiento no alcanza el nivel de una app nativa en casos de alta complejidad visual.
- Para produccion se debe fortalecer autenticacion, CORS y configuracion de ambiente.
