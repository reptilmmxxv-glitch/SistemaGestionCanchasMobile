# ADR-001: Framework Web

**Estado:** Aceptado  
**Fecha:** 2026-04-29  
**Decisor:** Equipo de Desarrollo  

## Contexto

La aplicación "Arriendo de Canchas" requiere un framework web robusto que permita:
- Gestionar modelos de datos complejos (canchas, reservas)
- Servir una interfaz de usuario mediante templates HTML
- Manejar solicitudes HTTP de forma eficiente
- Integrar una base de datos relacional

Se evaluó el mercado de frameworks Python disponibles considerando:
- Popularidad y comunidad
- Curva de aprendizaje
- Documentación
- Rendimiento
- Integración con bases de datos

## Opciones Consideradas

1. **Flask** - Microframework minimalista y flexible
2. **Django** - Framework full-stack con muchas características integradas
3. **FastAPI** - Framework moderno enfocado en APIs
4. **Bottle** - Un archivo simple, muy minimalista

## Decisión

**Usar Flask como framework web principal.**

## Rationale

1. **Flexibilidad:** Flask permite elegir componentes específicos sin imponer arquitecturas complejas
2. **Comunidad:** Comunidad grande y activa con amplia documentación
3. **Integración:** Flask-SQLAlchemy permite manejar la base de datos de forma elegante
4. **Curva de aprendizaje:** Ideal para estudiantes con excelente documentación educativa
5. **Escalabilidad controlada:** Para este proyecto de alcance medio, Flask es suficiente

## Consecuencias

**Positivas:**
- Flexibilidad en la arquitectura del proyecto
- Bajo overhead de dependencias
- Fácil de debuggear y entender para nuevos desarrolladores
- Compatible con Gunicorn para producción

**Negativas:**
- Menos características "out-of-the-box" que Django
- Requiere configuración manual para ciertos features
- Necesita librerías adicionales para autenticación avanzada

## Evidencia

En [app.py](../../app.py):

```python
app = Flask(__name__)
```
